# Mini-GPT с нуля на рассказах Чехова

Реализация языковой модели на архитектуре трансформера (GPT-style),
написанная с нуля на PyTorch — без использования готовых библиотек
трансформеров (Hugging Face и т.д.). Цель проекта — глубокое понимание
механики self-attention и архитектуры трансформера, а не production-решение.

## Что реализовано с нуля

- Посимвольная токенизация (character-level tokenizer)
- Self-attention (query/key/value, маскирование, масштабирование)
- Multi-head attention
- Блоки трансформера (attention + feed-forward + residual connections + layer norm)
- Позиционные embeddings
- Цикл обучения с оценкой train/val loss
- Генерация текста с управляемой температурой (temperature sampling)

## Архитектура модели

| Параметр | Значение |
|---|---|
| Параметров всего | 10.86M |
| Размер контекста (block_size) | 256 символов |
| Размер embedding (n_embd) | 384 |
| Количество голов attention (n_head) | 6 |
| Количество блоков трансформера (n_layer) | 6 |
| Размер словаря | 159 уникальных символов |

## Данные

Обучающий корпус — 77 рассказов А.П. Чехова (~1.8 млн символов),
собранных из открытого репозитория [d0rj/RusLit](https://github.com/d0rj/RusLit).

## Результаты обучения

- Train loss: 1.24 (после 5000 шагов)
- Val loss: 1.49
- Обучение на GPU (CUDA) — около [ВПИШИ СВОЁ ВРЕМЯ] минут

## Пример генерации (temperature=0.5)

```
Все перед положение ставились на старика Катинина и Павла Иваныча.
После просьбых стоял мне огонь, сидели в кармане и вошел в сторону,
с ней и на столе, прислушались с озеро.
```

## Пример генерации с затравкой ("Иван Иванович сказал")

```
Иван Иванович сказал:
– А как вы вы пошли?
– Да, у меня выходите из комнаты, не знаю, что вы, как бы я не люблю
и из этих слов, то понимаю, как приятных глуп.
```

Видно, что модель усвоила грамматику русского языка, пунктуацию
(включая диалоговые тире), стиль имён собственных, но не смысловую
согласованность на уровне целого текста — ожидаемое ограничение
для модели такого размера, обученной на таком небольшом корпусе.

## Как запустить

```bash
pip install torch

python 01_data.py          # подготовка данных и токенизация
python 04_gpt_model.py      # обучение модели (~[ВПИШИ ВРЕМЯ] на GPU)
python 05_generate.py       # генерация текста из обученной модели
```

## Структура проекта

```
├── input.txt              # обучающий корпус (рассказы Чехова)
├── 01_data.py              # токенизация и подготовка данных
├── 02_bigram.py            # baseline bigram-модель (для сравнения)
├── 03_attention_demo.py    # демонстрация механики self-attention
├── 04_gpt_model.py         # полная архитектура + обучение
├── 05_generate.py          # генерация из обученной модели
└── data.pt                 # сохранённые тензоры данных (создаётся 01_data.py)
```

## Что можно улучшить дальше

- Заменить посимвольную токенизацию на BPE (byte-pair encoding) —
  уменьшит длину последовательностей и может улучшить качество
- Обучить на большем корпусе (сейчас 1.8М символов — довольно мало)
- Добавить learning rate scheduling
- Эксперименты с большим количеством слоёв/голов attention
